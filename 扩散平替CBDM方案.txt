扩散平替CBDM方案
==================

一、最终目标
------------
本方案的唯一核心目标不是“换一个看起来不同的模块”，而是：

1. 用扩散模型或扩散思想，构造一个与当前CBDM在方法形态上不能太像的新模块。
2. 这个新模块要尽可能承担CBDM在DCBR中的真实作用，而不只是表面替代。
3. 最终目的始终是：平替CBDM，尽量保住DCBR在iFashion数据集上的指标涨点增益。

必须时刻记住：
我们不是为了“做一个扩散模块”而做模块，而是为了“保住CBDM带来的涨点”。


二、为什么之前的方向不够
----------------------
目前已经验证过的几条线，包括：

1. 静态权重矩阵筛边。
2. bottom-k / 百分位范围保留。
3. MBM（Masked Bundle Modeling）集合恢复。
4. Anchor Reconstruction Selector。

这些方法虽然都学到了一部分有用信息，但都没有完整吃下CBDM的增益。

当前阶段性判断是：

1. CBDM的收益不只是“选中了哪条top1边”。
2. CBDM更可能在做“全局结构先验 + 用户条件动态重排/生成”。
3. 如果只在离散边空间里静态选边，或者只学局部anchor支持关系，很难完整复刻CBDM的机制。

因此，后续新方案必须满足：

1. 仍然具有全局结构信息。
2. 仍然对单个用户的observed bundle集合做条件化处理。
3. 仍然是动态的，而不是静态边打分。
4. 与当前CBDM的“交互矩阵空间扩散”存在明显方法差异。


三、方案总览
----------
方案名称：
Set-Conditioned Latent Anchor Diffusion

简称建议：
SC-LAD

一句话定义：
先用LightGCN学出用户和bundle的UB结构表示，再把“用户observed bundle集合”的聚合latent表示送入扩散模块，在latent空间里生成一个“理想结构锚点表示”，最后只在该用户真实observed bundles中按照相似度选出top1或topk，用于重建UB图，替代CBDM。


四、与CBDM的本质区别
------------------
当前CBDM：

1. 输入对象是用户的UB交互向量。
2. 扩散发生在交互矩阵空间。
3. 输出是对所有bundle的重建分数。
4. 再从observed bundles中选top1/topk。

本方案：

1. 输入对象不是交互矩阵，而是用户observed bundle集合的latent聚合表示。
2. 扩散发生在latent空间，而不是交互矩阵空间。
3. 输出不是整行bundle分数，而是一个“理想结构锚点latent”。
4. 再从该用户真实observed bundles中做匹配选top1/topk。

因此，本方案和CBDM在下面三个维度上都不同：

1. 输入空间不同。
2. 扩散对象不同。
3. 输出形式不同。

这保证了本方案不是CBDM的简单改写，而是一个功能目标相近、技术形态不同的新模块。


五、方案想复刻CBDM的哪一部分能力
----------------------------
本方案并不是想复刻CBDM的全部形式，而是想复刻CBDM真正有价值的功能：

1. 在全局UB结构先验下处理单个用户。
2. 不进行静态边打分，而是做条件化生成/重排。
3. 每个epoch都可能给出不同的结构锚点。
4. 最终重建一个更有利于DCBR训练的UB结构图。

我们要复刻的是“功能与效果”，不是原始实现路径。


六、模块输入与前提
----------------
前提：
已经能够拿到基于iFashion_UB训练出来的LightGCN用户和bundle表示。

表示文件：

1. user_emb.pt
2. bundle_emb.pt

语义：

1. u_embed[u]：用户u的UB结构latent表示。
2. b_embed[b]：bundle b的UB结构latent表示。

这些表示是在全图上训练得到的，因此天然包含：

1. 全局协同信息。
2. bundle之间的结构关系。
3. 用户与bundle在全局UB图中的相对位置。

这一步为后续扩散模块提供全局结构先验。


七、核心流程
----------
Step 1：为每个用户构造observed set
--------------------------------
对用户u，记其训练集中真实observed bundles为：

O_u = {b1, b2, ..., bn}

这是当前用户的局部结构上下文。


Step 2：聚合用户的observed bundle latent
--------------------------------------
先将该用户observed bundle集合映射为一个初始集合表示：

z_u^0 = Pool({b_embed[b] | b in O_u})

第一版建议最简单：

z_u^0 = mean_{b in O_u}(b_embed[b])

后续可升级：

1. attention pooling
2. weighted pooling
3. prototype pooling

z_u^0 的含义：
它不是某个具体bundle，而是“用户当前observed bundle集合”的连续结构表示。


Step 3：构造条件输入
------------------
扩散模块不只看z_u^0，还要看用户侧结构信息。

第一版建议条件输入：

cond_u = [u_embed[u], z_u^0]

也可以后续加上：

1. 用户交互长度。
2. observed bundle统计特征。
3. prototype分布信息。


Step 4：在latent空间做扩散
------------------------
不再对原始UB向量扩散，而是对 z_u^0 做扩散：

q(z_u^t | z_u^0)

训练一个denoiser：

epsilon_theta(z_u^t, t, cond_u)

扩散模块学到的不是“哪个bundle分数高”，而是：

在全局UB结构先验下，如何把用户当前集合表示修正成更适合做结构锚点选择的理想latent表示。


Step 5：得到理想anchor latent
----------------------------
经过反向扩散，得到：

z_u^*

z_u^* 的含义：

1. 它不是某个真实bundle。
2. 它是该用户在当前epoch下的“理想结构锚点表示”。
3. 它是在全局结构先验和用户局部observed set共同作用下生成的。


Step 6：在真实observed bundles中匹配
---------------------------------
只在该用户的真实observed bundles中计算相似度：

score(u, b) = sim(z_u^*, b_embed[b]), 其中 b in O_u

第一版建议：

sim 使用 cosine similarity

然后：

1. top1_u = argmax score(u, b)
2. 或者保留 topk

再将所有用户的top1/topk边构造成新的UB图，送回DCBR主干训练。


八、训练目标设计
--------------
本方案不依赖CBDM标签。
不蒸馏CBDM top1，而是直接学“什么样的latent anchor对当前用户结构更有利”。

建议使用两个目标。


目标一：latent denoising loss
--------------------------
标准扩散损失：

1. 预测噪声。
2. 或直接预测clean latent。

记为：

L_diff

作用：
保证扩散模块能把噪声latent恢复为有结构意义的表示。


目标二：set consistency / anchor consistency
------------------------------------------
为保证 z_u^* 不偏离真实observed结构，需要一个辅助目标。

第一版可定义为：

L_set = - mean_{b in O_u} sim(z_u^*, b_embed[b])

含义：

1. z_u^* 应该与该用户observed bundle集合保持结构一致。
2. z_u^* 不应该漂到该用户结构分布之外。

后续可升级为：

1. top-m similarity consistency
2. prototype consistency
3. neighborhood consistency


最终目标：
--------
L = L_diff + alpha * L_set

alpha 为平衡系数。


九、为什么这个方案有希望平替CBDM
------------------------------
原因不是“也用了扩散”，而是它抓住了CBDM最可能真正有效的三件事。

1. 全局性
LightGCN的u,b表示是全图训练得到的，本身带有全局协同结构先验。

2. 用户条件性
输入不是单边，而是该用户的observed bundle集合聚合表示。

3. 动态性
扩散模块随训练更新，每个epoch都可能生成不同的理想anchor latent，因此最终top1/topk不是静态的。

这三点正是我们目前认为CBDM增益最可能的来源。


十、与当前几条替代路线的对比
--------------------------
1. 比权重矩阵筛边强
因为它不是静态边打分，而是动态latent生成。

2. 比MBM更对题
MBM本质上还是集合恢复模型；本方案直接生成理想anchor latent，更接近“结构锚点生成”。

3. 比当前anchor scorer更强
当前anchor scorer仍在离散边空间中做判别；本方案先生成连续锚点，再去匹配真实bundle，建模能力更强。


十一、第一版最小实现建议
----------------------
第一版务必简化，先做能跑、能验证机制的版本。

建议如下：

1. 使用现有LightGCN导出的 user_emb.pt 和 bundle_emb.pt。
2. Pool 方式用 mean pooling。
3. cond_u = [u_embed[u], z_u^0]。
4. denoiser 用小型MLP。
5. 扩散目标用标准latent denoising。
6. 辅助目标用 observed-set consistency。
7. 推理只在observed bundles里做 cosine matching。
8. rebuild_k 第一版先保持为 1。


十二、后续增强方向
----------------
若第一版有希望但差一点，再往这些方向升级。

1. attention pooling 替代 mean pooling
提升用户observed set聚合质量。

2. prototype diffusion
先把bundle聚成prototype，再对prototype mixture做扩散，进一步增强全局性并拉开与CBDM的差异。

3. residual diffusion
不直接扩散z_u^0，而是扩散残差：
r_u = z_u^* - z_u^0
使训练更稳。

4. top1 改 topk
如果发现单top1过硬，可以转向topk重建。

5. anchor-set support loss
让 z_u^* 不只接近 observed set 中心，而要更好地支撑 observed set 结构。


十三、潜在风险
------------
1. mean pooling 太粗糙，可能丢失集合内部结构。
2. z_u^* 可能退化成 observed set 的几何中心，而不是最优结构锚点。
3. 扩散模块可能学到“平滑表示”，但不一定能真正带来UB重建增益。
4. 最终top1虽然稳定，但未必有CBDM那种训练动态扰动效果。

因此，后续实验不能只看最终Recall/NDCG，还要看：

1. 选出的top1/topk分布。
2. 每个epoch的动态变化情况。
3. 与BOD后段区域的关系。
4. 是否确实在结构上比静态方案更像CBDM。


十四、实验目标与验收标准
----------------------
这个方案最终要回答的不是：

“有没有做出一个和CBDM不太像的扩散模型。”

而是：

“这个新模块能不能在不直接复制CBDM的前提下，替代CBDM在DCBR里的功能，并尽量保住iFashion上的涨点增益。”

验收标准建议按三层：

第一层：能稳定训练，能正常重建UB图。
第二层：重建行为比静态方案更像动态结构选择器。
第三层：最终Recall/NDCG尽量接近原始CBDM版本的DCBR。


十五、当前结论
------------
当前最值得推进的扩散替代方案，不是继续在交互矩阵空间复制CBDM，而是：

利用LightGCN提供的全局UB结构先验，在latent空间中对用户observed bundle集合做条件扩散，生成理想结构锚点表示，再从真实observed bundles中选出top1/topk重建UB图。

再次强调：
最终目标始终是平替CBDM并尽量保住涨点增益。
所有设计都必须围绕这个目标服务，而不是为了形式创新而创新。
