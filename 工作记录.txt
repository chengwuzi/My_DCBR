My_DCBR 工作记录

主题：
围绕 iFashion 数据集，研究如何平替或逼近 DCBR 中 CBDM 带来的指标涨点增益。


一、项目当前核心结论

1. 当前项目后续工作只关注 iFashion。

2. 原始 DCBR 的关键收益来源，越来越不像只是“选中了某条更好的 top1 边”，而更像：
   - 共享的全局去噪器
   - 对所有用户进行结构性动态重排
   - 训练期持续生成动态 UB 教师图

3. 已经有很强证据表明：
   - 我们目前各种替代方案已经能学到 U-B 交互的质量高低
   - 这也是为什么很多方案在 epoch4 第一次评估时效果异常好
   - 但大多数方案会在 epoch9 左右见顶，然后快速过拟合回落
   - 原始 CBDM 则能一直涨到 epoch20+，后期涨幅变小但仍持续刷新

4. 因此，当前最重要的判断是：
   - 问题不主要是“有没有学到边质量”
   - 而是“如何让结构教师图在 epoch9 之后仍然保持高质量、共享的、结构性的动态变化”


二、原始 DCBR / CBDM 训练逻辑的关键理解

1. 原始 DCBR 训练时：
   - 先训练 CBDM
   - 再用 CBDM 输出重建 UB 图
   - 再用该 UB 图参与 DCBR 主干训练

2. 原始 CBDM 在本项目中的作用更像：
   - 训练期结构教师图生成器
   - 而不是测试时直接负责最终召回

3. 测试阶段最终 user/bundle 表示主要来自 UI + BI 融合，UB 重建图不直接参与最终测试打分。
   这点在代码里已经核实：
   - train.py 里的 test() 调用 model.propagate(test=True)
   - DCBR.propagate(test=True) 返回的是 UI + BI 融合结果，不走 UB 图分支

4. use_observed_ub_rebuild_mask 的作用只是：
   - 在 CBDM 输出分数后，只允许从 observed train U-B 中选 topk
   - 它不改变 CBDM 本身在整行空间中的建模训练

5. BLCC 在 iFashion 上已经被消融实验验证影响极小，因此当前研究可以视为：
   - 重点研究 no-BLCC 情况下 CBDM 的核心机制

6. DCBR 主干训练本身一直存在图传播噪声正则：
   - UB/UI/BI 传播和聚合阶段都可加随机噪声
   - 所有方案都共享这部分机制
   - 因此它不能解释“为什么只有原始 CBDM 能持续涨到 epoch20+”


三、CBDM 选 top1 的统计结论

数据来源：
weightMatrix/DCBR-iFashion-noBLCC_observed_analysis-cbdm-epoch-top1.txt

1. 全训练阶段相邻 epoch 的 top1 变化情况：
   - 平均每个 epoch 约有 22416 个用户会更换 top1
   - 占全部用户约 41.6%

2. 后 10 个 epoch（20~29）：
   - 平均每个 epoch 约有 24268 个用户会更换 top1
   - 占全部用户约 45.0%

3. epoch 9~20 这个阶段更值得研究：
   - 平均每个 epoch 约有 48.26% 的用户会换 top1
   - 平均仍有约 11.45% 的用户会出现“之前没出现过的新 top1”
   - 这说明 9~20 期间不是简单微调，而仍然存在持续探索和重排

4. 后 10 个 epoch 中，每个用户的 top1 不是乱跳，而是在小候选池中切换：
   - 平均每用户只涉及 2.87 个不同候选
   - 中位数为 3
   - 43.21% 的用户只在 1~2 个候选中切换
   - 72.15% 的用户只在 1~3 个候选中切换

5. 结论：
   - CBDM 后期并没有静止
   - 但也不是无规律乱跳
   - 它更像是：先收缩到一个高质量候选池，再在这个候选池里持续进行共享的结构性动态重排


四、对 BOD 权重矩阵的研究结论

1. 文件：
   - weightMatrix/iFashion_UB.txt

2. 这不是 dense matrix，而是 observed train U-B edge list，每行是：
   - user_id bundle_id weight

3. 已确认：
   - 行数与 datasets/iFashion/user_bundle_train.txt 完全一致
   - 即它覆盖的是训练集 observed U-B 边

4. CBDM 选出的 top1 在 BOD 权重排序里的分布结论：
   - 整体明显偏向 BOD 排序后段
   - 但并不是永远等于倒数第 1 条
   - 更像是在后段一小片高质量候选区里动态游走

5. 已有的重要认识：
   - BOD 权重矩阵学到的不是“CBDM top1 的精确坐标”
   - 而是“CBDM 更可能出现的候选区域”


五、已经做过并验证过的方案

1. 静态 weight matrix 重建
   - use_weight_matrix_rebuild
   - use_weight_matrix_keep_bottomk_rebuild
   - use_weight_matrix_prune_bottomk
   结论：
   - keep-bottomk 是目前静态方案里最强的
   - 但静态图会在 epoch9 左右很快吃完收益，然后快速回落

2. bottom-k 上加噪声 / 加随机
   - 调大 upsilon_UB 没有效果，甚至下降
   - 在 bottom8 候选池内做随机选择，也没能复现 CBDM 的结构性变化
   结论：
   - CBDM 的换 top1 不是随机换
   - 而是共享全局去噪器驱动的结构性换边

3. MBMRebuilder
   - 文件：models/MBMRebuilder.py
   - 思路：masked bundle modeling + DeepSets
   结论：
   - 可以学到集合结构
   - 但更像“集合恢复器”，不是最贴近 CBDM top1 的机制

4. AnchorRebuilder
   - 文件：models/AnchorRebuilder.py
   - 思路：显式 anchor-conditioned reconstruction
   结论：
   - 比 MBM 更贴近“结构锚点选择”
   - 但仍然没能吃透 CBDM 的完整涨点

5. Weighted CBDM input
   - 开关：use_weighted_cbdm_input
   - 思路：保留原始 CBDM，只把 observed U-B 输入从 1 改为
     x(u,b)=max(eps, 1+gamma*(dot(u,b)-mean_u))
   - 其中 dot 为外部 LightGCN 的 raw dot product，mean_u 为用户自己的 observed 均值
   结论：
   - 方向合理
   - 但当前主线更值得继续压在 latent diffusion 上


六、当前最重要的主线：Latent Diffusion Rebuild

1. 核心目标
   - 用一个和 CBDM 形式不同的共享 latent diffusion 模块
   - 去平替 CBDM 的训练期 UB 动态教师图作用
   - 尽量保住 DCBR 的涨点增益

2. 与 CBDM 的区别
   - CBDM：在原始 UB 交互矩阵空间做扩散，输出整行 bundle 分数
   - Latent Diffusion：在 latent 空间做扩散，输出理想结构锚点 latent，再回到 observed bundles 里匹配 topk

3. Latent v1 当前实现
   - 文件：models/LatentDiffusionRebuilder.py
   - 训练接入：train.py
   - 外部 embedding：
     - weightMatrix/ifashion_gcn_emb/user_emb.pt
     - weightMatrix/ifashion_gcn_emb/bundle_emb.pt
   - 设计：
     - 冻结外部 LightGCN 的 u_embed / b_embed
     - 用 observed bundles 的 mean pooling 得到 z_u^0
     - 用共享 denoiser 在 latent 空间做去噪
     - 训练目标为：
       L = L_diff + latent_diffusion_set_loss_weight * L_set
     - 推理时用 z_u^* 与 observed bundle embeddings 做 cosine 匹配，选 topk

4. 目前对 latent v1 的理解
   - 它已经证明了“共享全局 latent diffusion 重排器”这个大方向是有效的
   - 但 v1 更像一个“单中心 latent 修正器”
   - 不如原始 CBDM 那么像“高维全空间、长链条的共享去噪重排器”


七、Latent Diffusion v1 的关键实验结果

实验设定逐步演进：

1. latent_diffusion_set_loss_weight=0.2, step=6, lr=1e-3
   - epoch4 test:  R@20=0.1089, N@20=0.0980
   - epoch9 test:  R@20=0.1425, N@20=0.1234
   - epoch14 test: R@20=0.1310, N@20=0.1099
   - epoch19 test: R@20=0.1316, N@20=0.1096
   结论：
   - 前期很强
   - 第二次评估后迅速塌掉

2. latent_diffusion_set_loss_weight=0.05, step=6, lr=1e-3
   - epoch14 test: R@20=0.1464, N@20=0.1229
   结论：
   - set_loss 调小有帮助
   - 但不是根本性突破

3. latent_diffusion_set_loss_weight=0.05, step=12, lr=1e-3
   - epoch4 test:  R@20=0.1089, N@20=0.0978
   - epoch9 test:  R@20=0.1429, N@20=0.1237
   - epoch14 test: R@20=0.1531, N@20=0.1283
   - epoch19 test: R@20=0.1546, N@20=0.1277
   结论：
   - 这是第一次明显突破 epoch9 这道坎
   - 说明 step 是真正影响后期增长形态的核心参数

4. latent_diffusion_set_loss_weight=0.05, step=12, lr=1e-4
   - epoch14 test: R@20=0.1532, N@20=0.1284
   结论：
   - 学习率从 1e-3 降到 1e-4，并未带来明显额外收益
   - lr 不是当前主矛盾

5. latent_diffusion_set_loss_weight=0.05, step=18, lr=1e-3
   - epoch14 test: R@20=0.1535, N@20=0.1285
   - epoch19 test: R@20=0.1546, N@20=0.1277
   结论：
   - step 从 12 提到 18，收益基本饱和
   - 说明当前 v1 的瓶颈开始更像结构瓶颈，而不是简单超参问题

6. topk=3 的尝试
   - loss_weight=0.05, step=12, lr=1e-3, topk=3
   - epoch19 test: R@20=0.1551, N@20=0.1310
   结论：
   - 这是目前 latent 线最好的结果
   - 已经非常接近原始 CBDM
   - 说明 latent 线已经真正看到希望


八、对目前 latent 结果的最新判断

1. 我们已经可以比较有把握地说：
   - latent 模块学去噪、学 UB 质量权重，肯定是能学的
   - 这条线不是方向错误

2. step 从 6 提到 12 后突然突破 epoch9 瓶颈，说明：
   - 当前 latent 与 CBDM 的差距，有相当一部分确实来自“扩散过程不够深、不够像真正的 diffusion 重排器”

3. 但 step 从 12 到 18 收益开始饱和，说明：
   - 现在剩余差距不再主要来自超参
   - 而更像当前 latent v1 的结构本身：
     - 单中心 mean pooled latent
     - deterministic clean prediction 迭代
     - observed cosine 匹配
   - 这些设计容易让后期重排器变成“高质量但偏静止的老师”

4. 当前阶段的判断：
   - latent v1 已经成功
   - 当前应先把一个强基线定住
   - 再考虑 latent v2 的结构升级


九、当前最推荐保留的强基线

对于 latent diffusion v1，目前最值得保留的强基线是：

- latent_diffusion_set_loss_weight = 0.05
- latent_diffusion_num_steps = 12
- latent_diffusion_lr = 1e-3
- rebuild_k = 3

这组是目前这条线最强、最接近 CBDM 的组合。


十、已经开发但当前优先级较低的功能

1. Latent reset 机制
   - 配置：
     - latent_diffusion_reset_after_epoch
     - latent_diffusion_reset_optimizer
   - 作用：
     - 到指定 epoch 后，只重置 latent 选择器（denoiser），不动 DCBR 主干和外部 embeddings
   - 当前状态：
     - 已开发
     - 当前优先级已降低，因为 step 的正向信号更强
     - 只有在 reset_after_epoch >= 0 时该功能才生效
     - reset_after_epoch = -1 时，无论 reset_optimizer 是 True 还是 False，都完全不起作用

2. MBM / Anchor / Weighted CBDM input
   - 都已开发并可运行
   - 但当前主线已经切到 latent diffusion，不再是优先路线


十一、对 CBDM 机制的阶段性总结

当前最重要的机制级判断：

1. CBDM 强的不是“会换 top1”本身，而是：
   - 共享的全局去噪器
   - 在整行高维 bundle 空间中进行重排
   - 所有用户在同一个 denoiser 下被共同重新排序

2. CBDM 后期依然能持续涨，不是因为一直发现全新 top1，而是因为：
   - 它在一个小的高质量候选池里持续进行结构性切换
   - 这种切换不是随机的，而是共享全局模型驱动的

3. static bottom-k 的问题更像：
   - 好图太快被当前损失吃透
   - 过早固定
   - 失去后续持续重排带来的收益

4. latent v1 之所以有效，是因为它首次开始复现：
   - 共享全局重排器
   - 用户条件性
   - 一定程度的后期动态性

5. latent v1 还没完全吃满 CBDM 涨点的原因，很可能是：
   - 当前 diffusion 动力学还不够“真”
   - 当前输出仍然太容易塌成“稳定中心 -> 稳定 topk”


十二、下一步最值得继续的方向

A. 短期继续实验
1. 先以当前最优的 latent v1 强基线为起点继续验证
2. 当前不建议再大范围扫很多小超参
3. 如果要继续小范围调，可优先围绕：
   - step=12 与 topk=3 附近的小范围组合
   - 看是否还能把 epoch19 往 epoch24 的节奏推近

B. 中期进入 latent v2
我认为 latent v2 最值得优先改的，不是继续磨 lr，而是结构升级：

1. 不再只用单个 mean pooled latent
   - 可以考虑多 slot latent
   - prototype mixture latent
   - set encoder 输出多个 token，而不是单中心

2. 让推理更像真正 diffusion
   - 不是只做 deterministic clean prediction 反复迭代
   - 而是更接近 posterior-style update
   - 甚至考虑适度保留 sampling noise

3. 让 bundle-level 结构关系更多参与最终打分
   - 不要完全压缩成一个中心再做 cosine

当前倾向：
   - latent v2 最值得先改的是“单中心 latent”这一点

十四、一句话总总结

现在最关键的阶段性结论是：

我们已经证明：
1. 只靠静态边质量不够；
2. CBDM 的核心更像共享全局去噪器驱动下的结构性动态重排；
3. latent diffusion v1 已经成功摸到这层机制，并通过增大 step 首次突破了 epoch9 的瓶颈；
4. 当前最有希望的后续方向，不是再回头大范围试旧 idea，而是以 latent v1 的强基线为起点，继续向更像“共享全局高维重排器”的 latent v2 进化。
