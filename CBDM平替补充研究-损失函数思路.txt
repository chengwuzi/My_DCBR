CBDM平替补充研究：从损失函数角度缓解 static bottom-k 快速过拟合
============================================================

一、这份补充文档的目的
----------------------
这份文档不是替代“扩散平替CBDM方案.txt”，而是作为并行补充。

当前我们已经形成两条值得保留的主线：
1. 扩散平替CBDM方案：尝试构造与CBDM形态不同、但功能上尽量承接CBDM作用的新扩散模块。
2. static bottom-k 改造方案：承认当前保留倒数top-k已经学到了不少u-b交互质量信息，转而重点研究为什么它会在epoch 9左右快速登顶然后过拟合回落。

这份文档聚焦第二条线。

始终记住最终目标：
不是为了调loss而调loss，而是为了尽量平替CBDM、保住DCBR在iFashion上的涨点增益。


二、当前现象与阶段性结论
------------------------
目前已有实验和统计基本支持以下判断：

1. 现有静态方案已经能学到u-b交互的质量高低。
   证据包括：
   - 很多方法在epoch 4第一次评估时效果异常亮眼，远高于原始DCBR在同阶段的表现。
   - 保留倒数top-k这条线长期都是当前最有效的静态替代方向。

2. 但这些方法的共同问题是：
   - 往往在epoch 9左右达到最佳。
   - 之后很快下降，出现明显过拟合。

3. 原始CBDM不一样：
   - DCBR + CBDM在iFashion上常常要到epoch 20甚至epoch 24之后还在涨。
   - 后期虽然涨幅变小，但没有像静态方案那样早早封顶。

4. 对CBDM epoch-top1记录的统计进一步说明：
   - 全程平均每个epoch约有41.6%的用户会更换top1。
   - epoch 9到20这段更活，平均约48.3%的用户每个epoch会换top1。
   - 这种变化不是随机乱跳，而是在一个小候选池里持续结构性切换。

5. 当前已经验证“随机动态”本身不是答案：
   - 单纯调大upsilon_UB没有用，甚至会下降。
   - 在bottom-k候选池里做简单随机切换也没有明显效果。

阶段性结论：
问题不在于“没学到边质量”，而更像是“静态好图在当前训练目标下被过早吃透了”。


三、为什么loss方向值得认真做
----------------------------
当前原始DCBR主干的训练目标可概括为：

Loss = BPR + CL

其中CL最关键的是三视图对齐：
1. UB <-> UI
2. UB <-> BI
3. UI <-> BI

代码位置：
- models/DCBR.py 中的 cal_cl_loss 与 forward

关键判断：
如果UB图是固定的bottom-k静态图，那么UB这条视图会变成一个“高质量但静态的老师”。

于是训练过程很可能变成：
1. 前期：
   UB图中的高质量结构信号被快速利用，所以epoch 4就很亮眼。
2. 中期：
   UI/BI越来越快地向这个静态UB老师靠拢。
3. 后期：
   静态老师已经没新东西可给，模型对其过度贴合，于是验证/测试指标开始回落。

因此，loss方向有依据的根本原因在于：
如果图本身短期内不会变，那就应该从训练目标上让UB老师的约束“释放得更慢、更软、更不容易被早早吃完”。


四、核心研究假设
----------------
当前最值得保留的工作假设是：

“static bottom-k图本身可能已经足够好，真正的问题不是图不对，而是当前损失函数让模型过快吸收并固化了静态UB老师的信号。”

换句话说：
当前瓶颈可能不是“再换更好的边”，而是“如何让静态老师图不要在epoch 9之前就把所有价值打完”。


五、最值得优先尝试的loss思路
----------------------------
以下方案按优先级整理。


方案A：将UB相关对比项单独拆出，并做时间调度（最推荐）
-----------------------------------------------------
当前inter-view对比项实际上绑在一起：
1. CL(UB, UI)
2. CL(UB, BI)
3. CL(UI, BI)

但这三项的作用并不对称。

研究判断：
最容易导致“静态UB老师被过快吃透”的，是：
1. UB <-> UI
2. UB <-> BI

而不是UI <-> BI。

因此，更合理的损失形式应改写为：

L = BPR
  + lambda_ub(t) * [CL(UB, UI) + CL(UB, BI)]
  + lambda_uibi * CL(UI, BI)
  + lambda_intra * IntraCL

其中：
1. lambda_ub(t) 是随epoch变化的动态系数。
2. lambda_uibi 固定。
3. lambda_intra 固定。

直觉解释：
如果静态UB老师一开始就被高强度对齐，很容易在前9个epoch就把价值耗尽。
而如果UB相关对齐是warmup式慢慢加强，就更像CBDM那种“教师图价值慢释放”的效果。

建议的第一版调度方式：
1. 线性warmup
   lambda_ub(t) = lambda_ub_max * min(1, epoch / T)
2. T 可先试 10 或 15
3. 也可以试分段常数：
   - epoch 0-5：较小
   - epoch 6-12：中等
   - epoch 13以后：正常强度

为什么优先级最高：
因为这条线最直接对应“静态图信号释放过快”这个问题本身。


方案B：对UB相关对齐加入“防过强贴合”约束
----------------------------------------
思路不是阻止对齐，而是防止过早、过强、过死地对齐。

可构造一个额外项：

L_cap = max(0, sim(UB, UI) - m_t)
      + max(0, sim(UB, BI) - m_t)

其中：
1. sim 表示某种视图相似度度量。
2. m_t 是允许的相似度上限，可为常数或随epoch增长。

作用：
1. 如果UB与UI/BI还没太像，不惩罚。
2. 一旦过早贴得太死，就开始约束。

直觉：
CBDM后期还能涨，一个重要现象就是UB教师图还在变，所以视图之间不会过早完全塌缩。
既然static bottom-k图不变，就用loss硬性保留一点“别那么快贴死”的空间。

优点：
1. 很对题。
2. 逻辑清晰。

缺点：
1. 需要定义好相似度阈值m_t。
2. 超参数解释稍复杂。


方案C：调大tau或对tau做时间调度，让CL前期更软
--------------------------------------------
当前对比学习温度tau较小，意味着对比学习更尖锐。
在static bottom-k图已经很有质量的情况下，较小tau可能会加速：
1. 视图快速拉近
2. 过快吃透静态图
3. 更早见顶

因此可以尝试：
1. 前期用较大tau
2. 后期逐渐降回原设定

例如：
1. epoch 0-9：tau = 0.15
2. epoch 10-19：tau = 0.12
3. epoch 20+：tau = 0.08

但这条线不如方案A直接，因为：
它会同时影响所有CL项，而不是只影响UB相关项。

定位：
更适合做辅助，不适合作为主线第一优先尝试。


方案D：提高UI-BI这对内部一致性的相对权重
----------------------------------------
思路是：
如果静态UB老师过强，模型会过度依赖UB视图；
那么可以适当提高UI-BI之间的自洽约束，让模型不要太快被UB带着跑。

形式上相当于：
1. 适当减弱UB相关inter-view项
2. 保持或稍增强UI-BI项

本质上是给UI/BI更多自主生长空间，避免它们过快被static UB老师同化。

这条线与方案A有一定关系，但更像其简化版。


六、当前最不建议优先做的事
--------------------------
基于现有结果，以下方向不建议继续作为主线优先尝试：

1. 单纯加大upsilon_UB
原因：
它影响的是特征扰动，不是结构性动态换边，也没有解决“老师图固定且被过快吃透”的根因。

2. 在bottom-k候选池里做简单随机切换
原因：
随机变化不等于结构性变化，已经验证效果很有限。

3. 继续大规模盲扫hidden size、dropout之类模型容量参数
原因：
当前更像目标和训练动力学问题，不像容量不够。


七、最建议的实验顺序
--------------------
如果后续重新接手这条线，建议实验按以下顺序推进：

阶段1：最小loss改造验证
1. 保留当前效果最好的static bottom-k方案（如倒数top8）。
2. 不改图，不改候选池，不加随机。
3. 只改loss，把UB相关inter-view CL拆出来，加入warmup。

首先只试这一件事，判断：
1. 最佳epoch是否从9向后推迟。
2. epoch 9之后是否不再快速掉。
3. 后期曲线是否明显更平缓。

阶段2：如果阶段1有信号，再细化调度
可继续试：
1. warmup长度 10 / 15 / 20
2. UB相关项起始权重更低
3. 逐步上调至原始强度

阶段3：如果阶段1仍不够，再加入“防过强贴合”约束
也就是方案B。

阶段4：最后才考虑辅助调tau
这一步放在后面，因为它更粗。


八、如果这条loss线成功，说明什么
-------------------------------
如果通过loss调度就能显著延缓epoch 9后的回落，说明：

1. static bottom-k图中包含的结构信息本身已经很有价值。
2. 问题主要在于训练动力学，而不主要在于边选错了。
3. CBDM相较于static方案的部分优势，可能确实来自“动态教师图慢释放”的效果。
4. 我们通过loss调度，部分模拟了这种慢释放。

这会是一个非常强的研究结论。


九、如果这条loss线仍然不够，说明什么
-----------------------------------
如果loss调度后仍然明显差于CBDM，那么更可能说明：

1. static图的确仍然缺少CBDM那种“结构性动态重排”本体；
2. 仅靠让老师信号慢释放，仍不足以完全模拟“图每个epoch都结构性变化”的作用；
3. 此时应回到扩散平替CBDM主线，继续围绕“共享全局重排器”推进。

换句话说：
这条loss线既有工程价值，也有研究价值。
不管成功还是失败，都会帮助更清楚地定位CBDM的真实增益来源。


十、当前最值得保留的总判断
--------------------------
这条思路线当前最重要的结论可以压缩成一句话：

“我们目前很可能已经学到了u-b交互质量高低，真正的问题不是图不够好，而是static好图在当前损失函数下被过早吃透；因此，与其继续做随机或加噪，不如优先从loss角度让UB教师图的约束慢释放、软对齐、别过早贴死。”

这句话应作为后续接手这条线时的总指导判断。
